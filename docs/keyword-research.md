# ブログ記事の方向性リサーチ手順書

> 更新日：2026-06-14

---

## 概要

次に書くブログ記事のテーマを「勘」ではなく「実際の検索需要」で決めるための手順。

- **Googleサジェスト**：世の中で実際に検索されている関連語（＝記事ネタ）を収集（認証不要・無料）
- **Search Console**：自サイトが既に表示されている検索語を確認（自サイトの現状把握）

両者を突き合わせ、「需要があり・競合が弱く・CVに近い」テーマを選ぶ。

---

## 起動コマンド（これだけ言えばOK）

```
Googleサジェストとサーチコンソールで今後の記事の方向性を調査して
```

Claude Code がスクリプト実行 → サジェスト＋SCデータ取得 → 分析 → 記事テーマ提案まで行う。

---

## 実行手順

### Step 1：初回のみ ── Pythonパッケージをインストール

```bash
pip install google-api-python-client google-auth
```
（サジェストは標準ライブラリのみで動作。SC取得に上記が必要）

### Step 2：スクリプトを実行

```bash
python -X utf8 automation/keyword-research.py
```

### Step 3：Claude Code で分析・記事テーマ提案

出力されたサジェスト・SCデータをClaudeに渡し、次の観点で分析する：

1. **需要の広さ** … 複数のシードで繰り返し出る語（例：「自作」が在庫・顧客・日報すべてで出る）は需要が広い
2. **CVへの近さ** … 「費用」「比較」「おすすめ」「発注」を含む語は購入意欲が高い
3. **競合の弱さ** … 業種×業務のロングテール（例：「食品 原価管理」）は競合が少なく上位を取りやすい
4. **ターゲット適合** … ITエンジニア向けの語（例：「仕様書 excel脱却」「excel 限界行数」）は除外する
5. **既存記事との重複（カニバリ）回避** … 自サイトの既存記事とキーワードが競合しないか確認

## Googleサジェストの仕組み（参考）

- エンドポイント：`http://suggestqueries.google.com/complete/search`
- `client=firefox` を指定すると `[入力語, [候補, ...]]` のクリーンなJSONが返る
- 認証不要・無料。ただし連続アクセスはブロックされうるため、リクエスト間に待機を入れている

## 検索ボリュームを正確に知りたい場合（発展）

サジェストは「何が検索されているか」はわかるが、**月間検索ボリューム（具体的な数値）は取得できない**。
ボリュームが必要な場合の選択肢：

| 手段 | 費用 | 備考 |
|------|------|------|
| Google Ads API（KeywordPlanIdeaService） | 無料 | 開発者トークン審査が必要。広告未出稿だと数値が範囲表示 |
| DataForSEO（従量） | 最低$50チャージ／1回数円〜 | サブスク不要。小規模に最適 |
| Keywords Everywhere | 約$1で1万キーワード | クレジット制 |

現状（初期サイト・記事ネタ探し）は **サジェスト＋Search Console で十分**。
ボリューム調査が継続的に必要になった段階で DataForSEO の導入を検討する。

## よくあるトラブル

| 症状 | 原因 | 対処 |
|------|------|------|
| サジェストが文字化け | PowerShellのエンコーディング | `-X utf8` をつけて実行 |
| サジェスト取得失敗（WARN） | 連続アクセス制限 | 時間をおいて再実行。`time.sleep` の値を増やす |
| SCがほぼ0件 | サイトが初期段階で表示が少ない | 正常。サジェスト側を主な判断材料にする |
| `403 Permission denied` | サービスアカウントがSCに未登録 | Search Consoleでアカウントを追加 |
