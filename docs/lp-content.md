# LPコンテンツ詳細設計

> **このファイルの役割**：「何を書くか」を定義するコンテンツ仕様書。確定テキスト・数字・セクション順序・FAQ回答文・CTA文言を管理する。デザイン（色・フォント・レイアウト）は `docs/design-template-1-guidelines.md` を参照。

## 設計意図
このドキュメントは、実装担当が `site/` へLPページを追加する際にそのままコピーして使えるコンテンツ設計をまとめたものです。
- 各ページごとにタイトル・見出し・本文・CTA・画像/リンクを定義
- ターゲット視点で訴求する（Excel管理の限界、中小企業課題）
- 漢那さん視点で更新しやすい構造を意識する
- Astro 実装時の `site/src/content/blog/` や `Formspree` の掲載位置も意識する

---

## 1. LPコンテンツの基本方針

### ターゲット
- Excel管理の限界を感じている中小企業の社長・責任者
- 物流・医療介護・EC・現場作業系などの業種
- 従業員20人前後の現場運営企業
- ITを投資と捉えられる人
- 継続的な保守・運用を理解できる人

### メイン訴求軸
- Excelやスプレッドシート管理は「一見安いが、実はコストが高い」
- 中小企業にこそ必要な「現場に寄り添う業務システム」
- 少数精鋭のチームだからこそ「無駄な管理コストを減らせる」
- 保守まで含めた「継続運用体制」
- まずは気軽に相談できる「フォーム接続」

---

## 2. LPページ一覧と役割

| ページ | 目的 |
|---|---|
| `/` | 入口。最短で相談につながる導線。実績と課題感を示して問い合わせ誘導。 |
| `/about` | ビズシスの信頼性を伝える。チーム紹介・実績・開発方針。 |
| `/services` | 提供サービスを具体化。何ができるか、何が得られるかを説明。 |
| `/cases` | 導入事例で信頼感を担保。業種別の成果ストーリー。 |
| `/tools` | ツール・技術力を示す。開発に使う技術や強みをやさしく説明。 |
| `/faq` | 不安を解消し、問い合わせのハードルを下げる。 |
| `/contact` | 問い合わせフォーム。Formspree で送信される。 |
| `/blog/` | 導線兼SEO。LPから関連記事として誘導。 |

### 3.6 `/faq` よくある質問

#### タイトル
- よくある質問

#### 質問例
1. 保守費用は必要ですか？
   - **回答**：はい。システムは作って終わりではなく、継続的な改善と安定運用が重要です。月1万円〜の保守プランで安心を提供します。
2. 開発期間はどれくらいですか？
   - **回答**：規模にもよりますが、簡単な業務システムであれば約1〜2か月程度が目安です。要件によって調整します。
3. Excel管理からシステムに移行したい場合、まず何をすればいいですか？
   - **回答**：まずは現状の運用と課題をヒアリングします。Excelの課題を整理したうえで、必要な機能を絞り込みます。
4. 料金はいくらですか？
   - **回答**：業務システム開発は30万円〜、保守は月1万円〜。機能・規模により変動しますので、まずはご相談ください。
5. 相談だけでもできますか？
   - **回答**：はい。まずはお気軽にご相談ください。文字ベースでのやり取りを基本としています。

#### CTA
- 「お問い合わせはこちら」ボタン（`/contact`）
- 「相談の流れを見る」リンク（`/contact` 下部へスクロール）

#### 画像/リンク

## 4. 実装担当への渡し方

### 4.1 渡す内容
- この `docs/lp-content.md` をそのまま実装担当に渡す
- `site/src/pages/` にページを追加し、上記タイトル・見出し・本文・CTA を反映する
- `site/src/content/blog/` に記事 Markdown を保存すること
- `site/src/pages/contact.astro` に Formspree フォームを設置する

### 4.2 実装上の注意点
- LP の構成は `7ページ + blog` を前提とする
- 各ページに常に `お問い合わせ` への CTA を設置する
- 画像はアイコンや図解を基本とし、過度な装飾は避ける
- 文章はターゲットの課題に寄り添い、問題解決の道筋を示す

### 4.3 運用性確保
- ページ構成は `site/src/pages/` に固定し、追加しやすい構造にする
- `site/src/content/blog/article-template.md` のテンプレートを活用し、新規記事作成を簡単にする
- `output/` にはビルド済み成果物と LP コンテンツの最終イメージを保存する
- `Formspree` の送信先メールは漢那さんのアドレスに設定し、テスト送信を実施する
