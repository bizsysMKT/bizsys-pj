## 2. PMから問われた2点への回答

### Q1：LP上部（hero → stats → team）の流れはどうなるか。「重い→重い→明るい」にならないか

**修正後の流れ：**

| セクション | 背景 | 視覚的印象 |
|-----------|------|----------|
| hero | ダークネイビーグラデーション | 重厚・インパクト大。訪問者を引きつける |
| stats | 白（明るい） | **開放感**。ダークから抜け出す「切り替わり」 |
| team | 白（明るい） | 読みやすい本文エリアへ自然に続く |

「重い→明るい→明るい」という流れになる。ヒーローで最大のインパクトを出し、そこから白背景の本文エリアへ切り替えることで、訪問者が「LP を読み進める」リズムが生まれる。2セクション連続ダークの圧迫感は解消された。

---

### Q2：グラスモーフィズムを除いた代替案はどうするか

**採用した代替案：白不透明カード＋強めのシャドウ**

```css
.hero-card {
  background: #ffffff;
  border-radius: 24px;
  padding: 32px;
  box-shadow: 0 20px 48px rgba(0, 0, 0, 0.25);
}
```

**この手法を選んだ理由：**

1. **可読性が最大**：白背景＋黒テキストは最も読みやすい組み合わせ。グラスモーフィズムの半透明＋白テキストより確実に視認できる
2. **ブラウザ互換性の問題なし**：`backdrop-filter` を使わないため、どの環境でも同一表示になる
3. **ダーク背景との対比が美しい**：白カードがダークグラデーションの中に浮かぶことで、視覚的なレイヤー感（奥行き）が生まれる。これはグラスモーフィズムより確実に「リッチ感」を出す手法
4. **「SaaSらしさ」からの脱却**：グラスモーフィズムはSaaS系LPの定番。廃止によって自らの制約（SaaSらしいUIは取り入れない）との矛盾が解消された

もう一つの選択肢だった「ダーク背景と調和する枠線カード（`border: 1px solid rgba(255,255,255,0.4); background: transparent`）」は、白テキストを読みやすくするために背景に十分なコントラストが必要になり、結局グラスモーフィズムと同様の問題（可読性・互換性）を抱える。白不透明カードの方が安全かつ効果的と判断した。
