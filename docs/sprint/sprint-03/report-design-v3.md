## 1. 決定した仕様の概要

### ① ヒーロー：パンチラインを1行で見せるレイアウト設計

**採用構造：3ゾーン構成**

```
[上段] .hero-top — eyebrow + h1（フル幅）
[下段] .hero-bottom — リード文+CTAボタン（左）| 課題共感カード（右）の2カラム
```

変更前の `.hero { display: grid; grid-template-columns: 1fr 1fr }` を廃止し、上段フル幅 + 下段2カラムの構造に分割した。

---

### ② ヒーロー：`border-radius` の方針

**採用値：`border-radius: 0`（フラット）**

上辺・下辺ともに角丸なし。`border-radius: 0 0 32px 32px` を廃止する。

---

### ③ サービスカード：アイコン仕様

**採用方法：インラインSVG（絵文字は不採用）**

| サービス | アイコン | 表現する価値 |
|---------|---------|------------|
| 業務システム開発 | モニター＋チェックマーク | 効率化・時間短縮・課題解決 |
| ツール・Webシステム開発 | 歯車 | 手作業削減・自動化・仕組み化 |
| 保守・運用サポート | シールド＋チェックマーク | 安心・継続性・長期パートナーシップ |

配置：各カード内 `<h3>` の直前、`display:block; margin-bottom:16px; width:40px; height:40px`。

## 2. 判断の根拠

### ① レイアウト：2カラム維持 vs 3ゾーン構成

**却下した選択肢**：
- `font-size: 3rem → 2rem` に縮小：パンチラインのインパクトが失われる。v2.0 で大きく設定した設計意図を損なう
- カラム比を `1.3fr 0.7fr` に変更：1200px コンテナで左列 ≈ 730px。22文字 × 48px（3rem）≈ 1056px の幅が必要で、比率調整だけでは解決しない

**採用した理由**：
3ゾーン構成はパンチラインをフル幅（約1200px）で表示できる。22文字 × 3rem の幅は約700px であり、コンテナ幅内に十分収まる。また「h1 が大きく単独で浮かぶ → 読者が受け取る → 下段に詳細が続く」という視線の流れを作れる。これはLP の情報設計としても自然で、課題共感のタイミングをコントロールできる。

---

### ② border-radius：フラット vs 小さいアール

**検討した選択肢**：
- `border-radius: 0 0 16px 16px`（小さいアール）：非対称は解消されないため却下
- `border-radius: 16px`（全辺均一）：フルページ幅のヒーローに均一アールを付けると、ページ内に浮いている印象になり「インパクトの薄さ」につながる

**採用した理由**：
B2B・専門サービス領域では、ファーストビューが画面幅いっぱいに広がりフラットな矩形になっているLPが多い（金融・SIer・企業向けSaaSなど）。フラットはカジュアルさを排除し、「誠実・安定・プロフェッショナル」を伝える視覚言語として機能する。中小企業の社長（40〜60代）のIT系サービス選定において、過度に柔らかいデザインは「信頼感が薄い」印象につながるリスクがある。

---

### ③ アイコン：インラインSVG vs 絵文字

**絵文字を不採用にした理由**：
- OS・ブラウザによって見た目が大きく変わる（Apple / Android / Windows で別デザイン）
- サイズ・色をCSSでコントロールできない
- ブランドカラー（`#0a4f9f`）への統一ができない

**インラインSVGを採用した理由**：
- stroke色 `#0a4f9f` で3アイコンを統一できる
- `.png/.jpg` 不要。実装担当がHTMLに直接貼り付けるだけで完結
- アイコンのテーマ（モニター / 歯車 / シールド）は業種・目的・安心感に直結するスタンダードなモチーフで、ターゲットに直感的に伝わる

## 4. 設計担当からPMへの確認事項

### A. `.hero-top` の `<h1>` 文字サイズについて（確認依頼）

3ゾーン構成でフル幅に h1 を置いた場合、`font-size: 3rem` だとパンチラインが短く見えすぎる可能性がある（逆に余白が広く取れすぎて「薄い」印象）。

**提案**：h1 をフル幅にするなら `font-size: 3.5rem`（デスクトップ）に大きくすることで、フル幅に対して適切な視覚的重量感が出る。

**PM確認依頼**：`3rem` を維持するか `3.5rem` に変更するかを確認してください。どちらでも実装担当が対応可能な範囲です。

---

### B. ヒーロー下段の `1fr 1fr` カラム比について（確認不要・設計判断として記録）

下段（`.hero-bottom`）の `1fr 1fr` は Sprint 2 から継続の比率。上段にフル幅の h1 が来たことで、下段のカードとリード文が「補足情報」として適切に後退する視覚的階層が生まれる。比率変更は不要と判断したが、PMの印象と異なれば再調整する。
