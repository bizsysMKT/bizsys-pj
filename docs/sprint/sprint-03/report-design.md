## 1. 更新内容の概要

`docs/design-template-1-guidelines.md` を v1.0 → v2.0 に全面改訂した。

### 何を、どの観点で変えたか

#### A. ヒーロー背景のダーク化（最優先・最高インパクト）

**変更前**：白背景、黒テキスト、白カード  
**変更後**：ダークネイビーグラデーション背景、白テキスト、グラスモーフィズムカード

**理由**：参考サイトとの「リッチ感の差」の最大要因がヒーローの視覚的インパクトの弱さにあった。ヒーローはLPで最初に目に入るセクションであり、ここをダーク＋グラデーションにするだけで全体の印象が大きく変わる。画像なしCSS のみで実現できる最大効果の変更。

#### B. 実績セクションのダーク化

**変更前**：白カード＋青数字  
**変更後**：ダークグラデーション背景＋白数字（ヒーローと同系統の配色）

**理由**：実績数字（14年 / 550件 / 89社）はビズシスの最大の信頼訴求材料。現状は白カードに埋もれていて視覚的な力がなかった。ダーク背景にすることで数字が際立ち、「実績の重さ」を演出できる。

#### C. フォント強化（Noto Sans JP 導入）

**変更前**：システムフォントのみ  
**変更後**：Noto Sans JP（wght 400/700/900）

**理由**：システムフォントは OS によって見え方が異なり、統一感が出ない。Noto Sans JP は日本語に最適化されており、wght 900 を hero h1 に使用することで見出しの重厚感と「リッチ感」が増す。

#### D. CTAボタンのグラデーション化

**変更前**：単色フラットボタン  
**変更後**：グラデーション＋シャドウ＋ホバー時の浮き上がり

**理由**：ボタンはユーザーのアクションを促す最重要要素。グラデーションとシャドウを加えるだけで立体感が生まれ、クリック意欲が高まる。CSS のみで実現可能。

#### E. セクションタイトルへのアクセントバー

**変更前**：テキストのみ  
**変更後**：左側にネイビーグラデーションのバー（`::before` 疑似要素）

**理由**：見出しの格が上がり、セクション間の視認性が向上する。1〜2行のCSS追加で実現できる費用対効果の高い変更。

#### F. カードのホバー効果追加

**変更前**：ホバー効果なし（静的）  
**変更後**：`translateY(-6px)` ＋ シャドウ強化

**理由**：静的なカードは「完成度が低い」印象を与える。ホバーアニメーションはサイトに生気を与え、リッチ感に直結する。

#### G. セクション背景の交互切り替え

**変更前**：全セクション白/淡いグレーで単調  
**変更後**：services・tools に `#f5f8ff`（淡いブルーグレー）を適用

**理由**：背景色が切り替わると「読んでいる場所の変化」が視覚的に明確になり、LPとしての奥行き・情報整理感が増す。

## 2. 参考サイトから取り入れた要素・取り入れなかった要素

### 取り入れた要素

| 要素 | 参考サイトの実装 | ビズシスへの適用 |
|------|----------------|----------------|
| ダーク背景セクション | ページ全体にコントラスト強いセクション | ヒーロー・実績セクションのみダーク化 |
| グラデーション | CTAボタン・背景に使用 | `.btn-primary` と hero 背景に適用 |
| フォントの多様なウェイト | Source Serif Pro（可変フォント）| Noto Sans JP の 400/700/900 |
| カードのインタラクティブ感 | ホバー効果・シャドウ | `translateY` ＋ shadow 強化 |
| セクション間の背景変化 | 複数の背景色でセクション分割 | 白と `#f5f8ff` を交互に |

### 取り入れなかった要素と理由

| 要素 | 理由 |
|------|------|
| **Serif系フォント**（Source Serif Pro） | ビズシスのターゲット（中小企業の社長・40〜60代）にはゴシック体の方が親しみやすい。Serif は洗練されすぎてITに距離感を感じるターゲットに馴染まない |
| **大型製品スクリーンショット・UI画像** | 実装担当は .png/.jpg を別途用意できない制約がある。SVG・CSSで代替できない規模の画像は対象外 |
| **企業ロゴの羅列（導入実績表示）** | 参考サイトはDENTSU等の大手企業ロゴを並べているが、ビズシスは守秘義務もあり実名ロゴは掲載できない。事例カードのテキスト形式を維持する |
| **過度なアニメーション**（スクロール連動・フェードイン等） | 中小企業の社長はITリテラシーが高くない場合もあり、動きが多すぎると「落ち着かない・信頼できない」印象につながるリスクがある |
| **SaaSらしいダッシュボードUI** | 参考サイトはSaaSのUIモックアップをビジュアルに使っているが、ビズシスは受託開発会社であり用途が違う |

## 3. PM確認が必要な判断ポイント

### A. ヒーローのダーク化について（要確認）

ヒーロー背景をダークネイビーグラデーションに変更する方針にしたが、これはLPの印象を「爽やか・明るい」から「重厚・専門的」に大きく変える変更。

**確認依頼**：依頼者（ユースケ）が期待する「リッチ感」の方向性が「重厚・専門的」か「洗練・スタイリッシュ（白基調のまま）」かを確認したい。

- 重厚路線：本ガイドの方針（ヒーロー・実績をダーク化）
- 白基調路線：カード・ボタン・フォントのみ改善し、ヒーローは白のまま

### B. Google Fonts の読み込みについて

Noto Sans JP は日本語フォントのため、ファイルサイズが大きい。ページ速度に影響する可能性がある。

**確認依頼**：パフォーマンスを重視するなら `display=swap` かつ必要ウェイト（400/700/900）に限定することを推奨する。`preconnect` を追加しているが、ページ速度テスト（Lighthouse）を実施するか否かを確認したい。

### C. グラスモーフィズムの `backdrop-filter` の互換性

`backdrop-filter: blur(8px)` は Safari では対応済みだが、古い Android Chrome では効かないことがある。

**確認依頼**：ターゲットの中小企業社長は PC 閲覧が多いと想定されるため大きな問題にはならないと判断したが、確認が必要であれば `backdrop-filter` を省略した代替仕様（半透明背景のみ）に変更可能。

## 4. Sprint 3 実装担当へのバトン

`docs/design-template-1-guidelines.md` の「優先度高」5項目を実装すれば、依頼者の「リッチにしてほしい」に対してインパクトの大きい改善ができる。

実装の推奨着手順：

1. Noto Sans JP の `Layout.astro` への追加（影響範囲全体、最初にやるべき）
2. ヒーローのダーク背景・テキスト白化（最大インパクト）
3. `.btn-primary` のグラデーション化（全CTAに一括適用できる）
4. 実績セクション（stats）のダーク化
5. `.section-title::before` でアクセントバー追加
