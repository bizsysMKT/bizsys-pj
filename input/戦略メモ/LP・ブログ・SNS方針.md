# LP・ブログ・SNS 方針

> Claude Codeとの壁打ちで決定した内容をまとめたもの。
> ユースケへの依頼資料の元ネタにする。

---

## 全体構成

```
独自ドメイン（既存）
  └── GitHub Pages
        ├── LP（トップページ）：問い合わせ獲得・信頼醸成
        └── ブログ：SEO目的の記事（Excelから業務システム化のテーマ等）
              ↑
        Claude Codeで記事の下書き生成・編集
              ↓
        GitHub Actionsで自動ビルド＆デプロイ
              ↓
        X（旧Twitter）に記事を自動シェア
```

---

## ① LP（ランディングページ）

### 目的
- ビズシスの実績・強みを見せる
- 問い合わせにつながる導線を作る

### 技術構成
- **GitHub Pages**（無料・独自ドメイン設定可・HTTPS自動対応）
- **Astro**（静的サイトジェネレーター）
  - markdownで記事管理できる
  - Claude Codeとの相性◎
  - テンプレートが豊富でデザインが整いやすい
- **問い合わせフォーム**：Formspree（無料プランで対応）

### コスト
- GitHub Pages：無料
- ドメイン：既存（biz-sys.jp）
- 追加費用なし

---

## ② ブログ

### 目的
- SEO：Googleで「Excel 限界 システム化」等のキーワードで見つけてもらう
- 信頼醸成：専門性・実績をコンテンツで伝える

### 技術構成
- Astroのブログ機能（LP と同じリポジトリで管理）
- 記事はmarkdownで書く
- Claude Codeで下書き生成 → 人が確認 → push → 自動デプロイ

### 想定するブログテーマ（例）
- 「Excelで管理していた引越業務をシステム化した話」
- 「訪問看護の記録管理をスプレッドシートから脱却した方法」
- 「業務システム開発の依頼前に知っておくべきこと」
- 「保守費用がかかる理由を正直に説明します」

---

## ③ SNS

### 方針
**今の段階でSNS運用に力を入れすぎない。ブログSEOを優先する。**

| SNS | 採用 | 理由 |
|---|---|---|
| X（旧Twitter） | ◎ 採用（自動シェアのみ） | ブログ記事の自動投稿で導線を作る。手動運用不要 |
| Facebook | ✕ 見送り | 有機リーチが激減・エンゲージメント管理が必要で継続困難 |
| note | ✕ 見送り | ターゲットの中小企業社長層に届きにくい・重複コンテンツリスク |
| スレッズ | ✕ 見送り | ユーザー層がターゲットとずれている・まだ早い |
| Zenn / Qiita | ✕ 見送り | 技術者向け。客取りには直結しにくい |

### X の使い方
- ブログ記事を公開したら自動でXに投稿（GitHub Actions）
- 手動でのSNS運用は不要
- フォロワー獲得は後から自然に育てる

---

## 優先順位

```
① LP を1枚作る（実績・強み・問い合わせ導線）
② ブログ環境を整える（Astro + GitHub Pages）
③ ブログ記事を5〜10本書く（SEOの種まき）
④ X への自動シェアを設定する
⑤ 効果を見ながら改善・追加
```

---

## 更新・運用の方針

| 作業 | 担当 | 方法 |
|---|---|---|
| LP・ブログの初期構築 | ユースケ | Astro + GitHub Pages のセットアップ |
| 記事の下書き生成 | Claude Code | markdownで自動生成 |
| 記事の確認・公開 | ユースケ → 漢那に引き継ぎ | 確認してpushするだけ |
| デプロイ | GitHub Actions | 自動（手動作業不要） |
| X への投稿 | 自動化ツール | 記事公開と連動して自動投稿 |
