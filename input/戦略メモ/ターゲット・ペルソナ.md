# ターゲット・ペルソナ整理

> この資料はClaude Codeとの壁打ちで整理した内容をまとめたもの。
> LP・ブログ・SNSの方向性を決める際の軸にする。

---

## ビズシスの理想形（大前提）

- 現在は少数精鋭（漢那・畑野の2名）
- 保守収入を増やして、ゆくゆくはメンバーをあと1〜2名増やしたい
- 精神的・肉体的・時間的に余裕を持ちながら活動したい
- 安定かつ着実に収益を積み上げたい
- 得意なシステム開発・保守を事業の軸にする

→ **「質の良いクライアントを積み上げて、保守収入を最大化 → メンバー追加で対応力を上げる」形が理想**

### 収益目標（暫定）
- 保守収入：月50万円以上（現状：月20〜23万円）
- 開発案件：1件30〜100万円（大型は単発でOK）
- 目標達成後、1〜2名採用を検討

## ターゲット像（ペルソナ）

**「Excel管理が限界になった、現場を持つ中小企業の社長・責任者」**

| 項目 | 内容 |
|---|---|
| 業種 | 物流・医療介護・EC・現場作業系など |
| 規模 | 従業員20人前後（さくらVPS 2GB・WAR4本運用のサーバスペックから逆算。1アプリあたり約400MBで快適に動く同時接続数の目安） |
| 課題 | Excel管理の限界・業務の属人化・特定業務の効率化 |
| 性格 | 自分のビジネスに責任感がある・ITに拒否感がない |
| 予算感 | 業務改善のための投資は惜しまない |
| 行動 | 自分でネットで調べて相談先を探す |

## 理想のお客さんの特徴

### ◎ こういう人がいい
- こちらの言葉や資料を理解する能力がある
- 開発者側の状況・苦労も感じ取ってくれる
- win-winになる方向に持っていこうとしてくれる
- 課題や解決策を丸投げせず、自分でも考えてくれる
- こちらの質問・疑問に主体的に答えてくれる
- レスポンスが早い・丁寧
- 保守費用の必要性を理解している（専門家に継続して任せたいと思っている）
- 資金に余裕がある

### ✕ こういう人は避けたい
- 丸投げする人
- レスポンスが遅い（1週間に1回、追加連絡しないと返事しない）
- こちらの質問を無視して自分の言いたいことだけ言う人
- 自分の要望だけを通そうとする人
- システムを「買い切り」だと思っている
- できるだけ安くやりたいだけで予算・投資意識がない

## 本質的なターゲット軸

**「ITをコストではなく、投資だと理解している人」**

- IT投資の価値を自分で考えられる
- 相手の専門性にリスペクトがある
- 自分のビジネスに責任を持っている
- 維持・継続の必要性を理解している

---

## 「Excel限界系」がベストターゲットである理由

- 課題が明確 → 何を作ればいいかわかりやすい
- 自分で限界を感じているので費用に納得しやすい
- 保守の必要性も理解しやすい（Excelが壊れたときの怖さを知っている）
